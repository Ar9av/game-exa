# Example games

Three complete Phaser 3 games generated by gameforge — one per validated genre. Each one ships with its full source, sprite sheets, level data, game-state.json, and a `qa/__baselines__/` set so you can run `npm install && npm run dev` and play immediately.

| Folder | Genre | Win condition | Lose condition |
|---|---|---|---|
| `slime-slayer/` | Top-down adventure | Collect 3 gems | HP reaches 0 |
| `pixel-pete/` | Platformer | Collect 5 coins | HP 0 or fall off screen |
| `star-defender/` | Shoot-em-up | Destroy 10 asteroids | HP reaches 0 |

## Run any example

```bash
cd examples/pixel-pete       # or slime-slayer / star-defender
npm install
npm run dev                  # opens http://127.0.0.1:5173
```

Controls:

- **Movement**: arrow keys or WASD
- **Action** (attack / jump / fire): Space

## Run the QA harness against an example

From the repo root:

```bash
node skills/playtester/scripts/run_qa.mjs examples/pixel-pete
```

This boots a headless Chromium, runs the per-genre scenarios, diffs against `qa/__baselines__/`, and writes `qa/qa-report.json`.

## Visual notes

These examples use **procedural placeholder sprites** (flat-color cells with state-specific tints), not the fal.ai GPT-Image-2 output. Procedural sprites are what gameforge generates without `FAL_KEY` set or with `--placeholder-sprites` passed.

To regenerate with real pixel-art sprites, set `FAL_KEY` and re-run:

```bash
cd examples/pixel-pete
gameforge generate --skip-design --skip-arch  # (regenerate sprites only — coming in v0.2)
```

Or simply re-run the full pipeline with the same prompt and a key set.

## How they were built

Each example was generated by the framework's deterministic pipeline (no LLM calls), using a hand-crafted GDD/levels/code in `test/fixtures/<name>.js`. The fixtures match what a real LLM agent would produce given the genre prompt — they exercise every code path the live agents use.

To regenerate any example from its fixture:

```bash
node test/run-fixture.mjs slime-slayer 5180
```

This wipes `/tmp/gameforge-test/slime-slayer/`, rescaffolds, regenerates assets, runs QA, and reports.
